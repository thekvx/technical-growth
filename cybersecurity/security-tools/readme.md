# 🧰 Security Tools

Security tools help provide visibility into systems, networks, and user activity so that suspicious behavior can be identified, investigated, and understood. Learning how these tools work is an important part of building practical cybersecurity skills.

This section documents the security tools I've studied, the practical skills I'm developing, and the hands-on experience I'm gaining through certification studies, labs, and continued practice. Everything included here reflects tools I've personally used or explored as I continue building the knowledge needed for a career in cybersecurity.

---

# 📚 Areas of Study

### 📊 Security Information & Event Management

Building practical experience with SIEM platforms such as Splunk to search logs, review events, create dashboards, and better understand how security teams monitor activity.

---

### 🌐 Network Analysis

Learning how tools such as Wireshark and tcpdump provide visibility into network traffic and support troubleshooting, investigation, and security analysis.

---

### 🔍 System & Network Discovery

Developing familiarity with tools such as Nmap to identify systems, services, ports, and other information that can help build a clearer understanding of an environment.

---

### 🧪 Security Testing

Exploring tools used in guided labs to better understand common weaknesses, attack methods, and the defensive controls used to reduce risk.

---

### 📝 Analysis & Documentation

Practicing how to interpret tool output, record findings, and explain what the available evidence shows without relying on the tool alone to provide the answer.

---

# 💡 Learning Philosophy

Security tools are most useful when the person using them understands the systems, traffic, and activity being analyzed. Rather than treating tools as shortcuts, I focus on learning what their output means, why it matters, and how it connects to the larger investigation.

This section isn't intended to be a complete reference for security tools. Instead, it documents the tools I've personally worked with and the knowledge I'm building through continued hands-on practice.
